# My SQL

## Understanding Database Relationships

* Child Table and Parent Table:
  The **child** table which references to the **parent** table using _foreign key_.

* Primary and Foreign key:
  **Primary** key exists to uniquely indentify a entry(row) in a table.
  **Foreign** key exists to reference, that is to store primary key of _parent table_.

* Relationships between table entries:
  1. <u>One-to-One Relationship</u>
     When **only one** _entry_ of a child table references to **only one** _entry_ of a parent table.

  2. <u>One-to-Many Relationship</u>
     When **many** _entry_ of a child table reference to **only one** _entry_ of a parent table. 

  3. <u>Many-to-Many Relationship</u>
     Here, we create a third table, a _common_ child table.
     This references to **many** _entries_ from two different _parent_ tables. (muliple foreign keys in one table).
     This _common_ child is also known as **junction table**.


## ERD (Entity-Relationship Diagrams)

 *  Elements
    1. Entities => Tables
    2. Attributes => Columns
    3. Relationships => Connection between tables.
    4. Cardinality => Relationship (1-1, 1-n, n-n).
